HTML5
HTML5 is a markup language used for structuring and presenting content for the Internet and it aims to improve the language with support for the latest multimedia while keeping it easily readable by humans and consistently understood by computers and devices.

Introduction to HTML5
  • HTML5 is the proposed new standard for HTML 4.01, XHTML 1.0, and DOM (Document Object Model).
  • It was developed by the World Wide Web Consortium (W3C).
  • It aims to reduce the need for proprietary plug-in-based Rich Internet Application (RIA) technologies such as Adobe Flash and Microsoft Silverlight
Vocabulary
  • HTML - stands for Hyper Text Markup Language. Hypertext Markup Language is the markup language that was used to build many of the first sites on the Web.
    • Text - a sequential form of characters in human understandable form. Hyper Text - text which is not constrained to be linear (because it contains links to other text). Markup Language – uses tags, or special sequences of characters, to tell a program how to interpret a text file. Examples include HTML, XHTML, XML, HTML5 Tag – tells a browser how and where to display text and images on a Web page. Text in a tag appears between two angle brackets (<>). This tells the Web browser how to display a pages content. Start Tag – a tag that begins the code for a function in markup languages. It is denoted with the angle brackets (<>). End Tag – a tag that ends the code for a function in markup languages. It is denoted with the angle brackets and a backslash after the open angle bracket (). HTML5 - an updated markup language for structuring and presenting content for the World Wide Web and a core technology of the Internet. It was started by Mozilla, Apple, and Opera under a group called the WHATWG - Web Hypertext Application Technology Working Group. Attribute - a modifier of an HTML element that generally appears as name-value pairs, separated by an "=", and are written within the start tag of an element, after the element's name. Example: <tag attribute="value">(content to be modified by the tag)</tag>
Timeline of HTML5
  • 1991 - HTML
  • 1994 - HTML 2
  • 1996 - CSS 1 + JavaScript
  • 1997 - HTML 4
  • 1998 - CSS 2
  • 2000 - XHTML 1
  • 2002 - Tableless web design
  • 2005 - AJAX (Asynchronous JavaScript and XML) - A web technique used to send data to, and retrieve data from, a server asynchronously (in the background) without interfering with the display and behavior of the existing page.
  • 2007 - HTML5 started by Mozilla, Apple, and Opera under a group called the WHATWG - Web Hypertext Application Technology Working Group.
  • 2010 - Although HTML5 has been well known among web developers for years, it became the topic of mainstream media around April 2010 after Apple Inc's then-CEO Steve Jobs issued a public letter titled "Thoughts on Flash" where he concludes that "[Adobe] Flash is no longer necessary to watch video or consume any kind of web content" and that "new open standards created in the mobile era, such as HTML5, will win". This sparked a debate in web development circles where some suggested that while HTML5 provides enhanced functionality, developers must consider the varying browser support of the different parts of the standard as well as other functionality differences between HTML5 and Flash.
  • 2011 - Adobe announced that it will discontinue development of Flash for mobile devices and reorient its efforts in developing tools utilizing HTML5.
  • 2011 - The WHATWG renamed its "HTML5" living standard to "HTML". The W3C nevertheless continues its project to release HTML5. The concept of a living standard is that it is never complete and is always being updated and improved. New features can be added but functionality will not be removed.
  • 2014 - W3C plans to release a stable HTML5 Recommendation by the end of 2014
  • 2016 - W3C plans to release a stable HTML 5.1 specification Recommendation by the end of 2016
Advantages of HTML5
  • Good support on modern mobile devices:
    • iOS
    • Android
    • Windows
  • Simpler, more intuitive syntax
  • Video and audio without requiring a plugin
  • Incremental improvements to previous HTML challenges
  • Creative enhancements
  • Rounded corners
  • Gradients
  • Text layout
  • Emergence of more advanced technologies such as O3D (WebGL)
  • O3D is an open source JavaScript API created by Google for creating interactive 3D graphics applications that run in a web browser window
  • O3D may be crafted for use in any application area, however it is geared towards games and advertisements
  • Easier SEO (Search Engine Optimization)
  • Search Engine Optimization is the process of affecting the visibility of a web page in a search engine’s “natural” or un-paid (“organic”) search results
